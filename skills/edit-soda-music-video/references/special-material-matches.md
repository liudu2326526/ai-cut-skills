# 特殊素材匹配规则

特殊素材匹配是普通 `description` 语义排序之前的强制覆盖规则。是否命中以及素材画面是否符合要求，都由执行 Skill 的模型阅读当前口播、Read 当前素材后判断；脚本不根据台词自动选材，也不校验素材 SHA-256，只校验声明的规则 ID 是否存在。

特殊规则仍必须满足渠道、金额和利益点授权。命中后优先使用下面的参考素材；如果素材经过重命名、重新压缩、合理缩放、轻微裁切或增加不遮挡关键信息的标注，只要执行模型确认核心画面仍完整，就可以继续使用。余额卡片、提现成功页或缺失关键档位的局部截图仍不符合本规则。

## `withdraw_0_3`：满三毛提现

### 触发条件

同一条利益点表达必须同时包含以下两层语义：

1. 门槛金额明确是三毛，即 `三毛`、`三毛钱`、`0.3 元`、`0.30 元` 或 `零点三元`；
2. 明确表达达到该门槛后可以提现，例如“满三毛就能提现”“攒够三毛可提现”“满 0.3 元即可提现”。

只出现“三毛”“0.3 元”“余额 0.3 元”，或只泛称“可以提现”，不得触发。`15 元`、`30 元`、其他提现额度和提现成功结果页不得触发本规则。

### 参考素材与画面要求

- 规则 ID：`withdraw_0_3`
- 素材库相对路径：`正文端内物料/金币音乐新/提现.png`
- 文件名：`提现.png`
- 源尺寸：`1080×1920`
- 画面事实：完整提现页面，顶部余额为 `8.72 元`，下方两组提现账户都展示 `0.3 元 / 15 元 / 30 元`档位。
- 推荐 Manifest 描述：`完整的提现页面截图，顶部显示余额 8.72 元，下方两组账户均提供 0.3 元、15 元和 30 元提现档位，可准确表达满 0.3 元即可提现的利益点。`
- 源像素有效区域：`x=99, y=105, width=882, height=1743, coordinate_space=source_pixels`

用户提供的局部截图只用于帮助定位素材，不是优先使用的正式资产。素材应保留提现页标题、余额区域和包含 `0.3 元`的提现档位；允许存在不遮挡关键信息的红框等轻量标注，但不得用只剩局部档位的截图或 `可用余额-0.3.png` 代替完整提现页。

如果任务的 `asset_root` 就是 `汽水物料-新`，优先使用上面的相对路径。如果 `asset_root` 更高一级，或素材已经被重命名、压缩、裁切或标注，则由执行模型在 Manifest 中根据 `description` 找到候选，再用 Read 确认仍是完整的 `0.3 元`提现页，时间轴写入当前 Manifest 的真实 `relative_path`。素材必须已被 `sync-assets` 收录，并完成 `description` 与 `effective_region` 理解。

### 时间轴写法

```json
{
  "name": "满三毛提现页",
  "category": "benefit_visual",
  "path": "正文端内物料/金币音乐新/提现.png",
  "kind": "image",
  "layout": "phone",
  "semantic_role": "benefit_point",
  "matched_benefit_text": "满三毛就能提现",
  "special_match_rule": "withdraw_0_3",
  "time_mode": "input",
  "start": 0.0,
  "end": 1.0
}
```

入点和出点必须直接使用 Whisper 对当前 input 识别出的字幕 `start/end`，并标记 `time_mode=input`。素材从“满/攒够/达到 0.3 元”这一利益点开始出现，覆盖到“提现”语义说完即可；不要提前覆盖前一句非利益点，也不要延长到下一个无关利益点。

## 执行顺序

1. 执行模型逐句判断是否命中特殊规则；
2. 命中时优先选择参考路径；路径不存在或素材已变体时，根据 description 找候选并用 Read 确认完整画面；
3. 在时间轴同时写入 `semantic_role`、`matched_benefit_text` 和 `special_match_rule`；
4. 运行 `preflight` 确认素材已入 Manifest、理解字段完整且规则 ID 合法；SHA-256 不参与校验；
5. 未命中特殊规则的其他利益点再按 Manifest `description` 做普通语义匹配。
