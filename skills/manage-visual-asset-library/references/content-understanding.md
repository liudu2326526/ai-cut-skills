# Read 素材理解与有效区域标注

## 原则

执行 Skill 的模型就是视觉理解模型。直接使用 Read 或可用的图像查看工具查看原图和视频代表帧；不要调用素材理解脚本、额外模型 API 或向量服务。脚本可以扫描文件、探测媒体和导出代表帧，但不能替模型描述画面。

不能只看文件名、目录名、粗分类或视频第一帧。无法从画面确认的事实不得写入 description。

## 图片流程

1. 从 Manifest 找到新增、修改或缺少理解字段的图片。
2. 使用 Read 打开原图，按真实分辨率确认主体、文字、数字、状态和留白。
3. 透明 PNG 必要时分别放在浅色与深色背景上查看，区分透明留白和实际内容。
4. 写入准确中文 description。
5. 把人物、UI、文字、图标、按钮及具有表达作用的背景合并为最小矩形 effective_region。
6. 透明留白、纯色无内容边距和空画布不进入 effective_region。

## 视频流程

1. 使用 `extract_video_frames.py` 导出首帧、四分之一、中点、四分之三和接近尾帧的代表帧；短视频会自动去除重复时间点。
2. 使用 Read 查看全部代表帧，确认画面变化、操作步骤和最终状态。
3. description 覆盖视频中真实出现的主要页面、动作和状态变化，不能只描述第一帧。
4. effective_region 使用全部代表帧有效内容的联合边界。
5. 如果代表帧不足以确认快速变化内容，可额外导出需要的时间点并继续 Read。

## description 写法

通常使用 `2–4` 句具体中文，覆盖适用信息：

1. 素材或页面类型：完整页面、弹窗、结果页、卡片、图标、照片或操作录屏；
2. 主体和布局：人物、账户、分组、卡片、按钮或档位的数量与排列；
3. 关键可见文字与数字：金额、时长、次数、比例、名称和状态值；
4. 动作与当前状态：正在执行什么，当前为选中、成功、失败、可用或禁用；
5. 画面可直接支持的语义：只写视觉证据能够表达的功能、场景或结果；
6. 易混淆时的排除信息：例如“不是结果页”“不是单独余额卡片”。

内容单一的图标或装饰素材可以只写一句；手机 UI、金额、权益、步骤操作等信息密集素材必须完整记录结构、文字、数字和状态。不要使用“多个按钮”“一些金额”“展示功能”等可以被真实信息替代的空泛表达。

局部文字无法辨认但不影响素材身份时，明确说明“部分文字不可辨认”。关键页面类型、数字或状态无法确认时，保持对应理解字段缺失并报告人工确认，不得根据文件名补全。

## effective_region 写法

```json
{
  "x": 440,
  "y": 860,
  "width": 200,
  "height": 200,
  "coordinate_space": "source_pixels"
}
```

例如 1080×1920 文件只有中心 200×200 图标承载内容，就只记录该区域，不把整张空画布视为有效内容。有效区域只描述源素材事实，不负责消费项目的安全区、位置、缩放或裁切策略。

## 写回前自检

- 是否实际 Read 过原图或全部代表帧；
- 是否写明具体素材类型、主体、布局和数量；
- 是否逐项记录清晰可见的关键文字和数字；
- 是否说明动作、阶段和当前状态；
- 是否只写画面能直接支持的语义；
- 是否需要排除相似页面；
- effective_region 是否排除了透明或无内容区域；
- 坐标是否来自源像素且没有越界；
- 是否存在只凭文件名推断或空泛概括。

任何一项不满足时重新 Read 并修正，不能直接进入语义检索。
